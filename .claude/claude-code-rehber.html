🔒 Hooks — Otomatik Güvenlik ve Otomasyon

Claude ne yaparsa yapsın, bunlar deterministik olarak çalışır

📁 ~/.claude/settings.json

⚠️ Önemli fark: CLAUDE.md'ye "rm -rf kullanma" yazarsan Claude bunu unutabilir. Ama hook koyduysan, fiziksel olarak bloke edilir. Fark bu.

❌ HOOK OLMADAN

Claude kazara .env dosyasını düzenledi. API anahtarların değişti. Production çöktü. Fark ettin mi? Hayır.

✅ HOOK İLE

.env'ye dokunmaya çalıştığında sistem "ENGEL: Bu dosyaya izin yok" diyerek durdurur. Otomatik. Her zaman.

~/.claude/settings.json — Ev dizinine koy, TÜM projelerde çalışır 
{
  "hooks": {
    // KURAL 1: .env ve gizli dosyalara dokunma engeli
    "PreToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [{
          "type": "command",
          "command": "if echo \"$CLAUDE_TOOL_INPUT\" | grep -qE '\\.env|\\.secret|credentials'; then echo 'ENGEL: Hassas dosyaya erişim yasak!' && exit 2; fi"
        }]
      },
      // KURAL 2: rm -rf gibi tehlikeli komutları durdur
      {
        "matcher": "Bash",
        "hooks": [{
          "type": "command",
          "command": "if echo \"$CLAUDE_TOOL_INPUT\" | grep -qE 'rm -rf|sudo rm|dd if='; then echo 'TEHLIKE: Bu komut onay gerektiriyor!' && exit 2; fi"
        }]
      }
    ],
    // KURAL 3: Dosya düzenlenince otomatik formatla
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [{
          "type": "command",
          "command": "prettier --write \"$CLAUDE_FILE_PATHS\" 2>/dev/null || true"
        }]
      }
    ],
    // KURAL 4: İş bitince masaüstüne bildirim gönder (Mac)
    "Stop": [
      {
        "hooks": [{
          "type": "command",
          "command": "osascript -e 'display notification \"Claude işi tamamladı ✅\" with title \"Claude Code\"' 2>/dev/null || echo 'Bitti!'"
        }]
      }
    ]
  }
}
.claude/settings.json — Proje klasörü içine koy, SADECE bu projede çalışır 
{
  "hooks": {
    // Bu projeye özel: TypeScript dosyası düzenlenince lint çalıştır
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [{
          "type": "command",
          "command": "if echo \"$CLAUDE_FILE_PATHS\" | grep -q '\\.ts'; then npx eslint \"$CLAUDE_FILE_PATHS\" --fix 2>/dev/null || true; fi"
        }]
      }
    ]
  }
}

⚡ Skills — Özel Slash Komutları

Markdown dosyasına yaz, her projede /komut olarak kullan

📁 ~/.claude/commands/
❌ SKILLS OLMADAN

"Git diff'e bak, değişiklikleri analiz et, uygun bir commit mesajı yaz, conventional commits formatında, sonra commit at" — Her seferinde bu uzun promptu yazıyorsun.

✅ SKILLS İLE

/commit yazıyorsun. Claude tüm bu adımları otomatik yapıyor. 2 saniye.

~/.claude/commands/commit.md — Bu dosyayı oluştur 
# /commit — Akıllı Git Commit
# Bu komutu kullanmak için: Claude Code'da /commit yaz

Aşağıdaki adımları sırayla yap:

1. `git diff --staged` çalıştır, staged değişiklikleri gör
2. `git diff` çalıştır, unstaged değişiklikleri de kontrol et
3. Değişikliklerin amacını ve kapsamını analiz et
4. Conventional Commits formatında commit mesajı oluştur:
   - feat: yeni özellik
   - fix: hata düzeltme  
   - refactor: kod düzenleme
   - docs: dokümantasyon
   - test: test ekleme
5. `git add .` çalıştır
6. `git commit -m "[mesaj]"` ile commit at
7. Ne commit ettiğini Türkçe özetle
~/.claude/commands/review.md — Kod inceleme komutu 
# /review — Kapsamlı Kod İnceleme
# Kullanım: /review src/components/Login.tsx

$ARGUMENTS dosyasını 5 açıdan incele ve Türkçe rapor ver:

1. **Güvenlik**: SQL injection, XSS, auth açıkları var mı?
2. **Performans**: Gereksiz re-render, N+1 query, memory leak?
3. **Kod Kalitesi**: DRY prensibi, okunabilirlik, isimlendirme
4. **Test Edilebilirlik**: Mock'lanabilir mi, side effect'ler ayrılmış mı?
5. **TypeScript**: any tipi var mı, tipler doğru mu?

Her bölüm için: ✅ İyi / ⚠️ Dikkat / ❌ Kritik sorun işareti koy.
Sonunda öncelikli düzeltme listesi ver.
~/.claude/commands/deploy.md — Deploy komutu 
# /deploy — Güvenli Deploy Akışı

Aşağıdaki sırayı ADIM ADIM takip et, her adım başarılı olmadan devam etme:

1. `npm run test` → Testler geçiyor mu? Geçmiyorsa DUR, düzelt.
2. `npm run build` → Build alınıyor mu? Hata varsa DUR, düzelt.
3. `npm run lint` → Lint hataları var mı? Kritik olanları düzelt.
4. Son değişiklikleri özetle, deploy edilecek şeyi göster
5. Onayımı iste: "Deploy etmemi onaylıyor musun?"
6. Onay verince `git push origin main` çalıştır
7. Sonucu rapor et

📋 Rules — Global Kurallar

Bir kere yaz, tüm projelerde geçerli

📁 ~/.claude/CLAUDE.md

Proje CLAUDE.md'den farklı: bu ev dizininde, tüm projelerde geçerli. Genel çalışma stilini buraya yaz.

~/.claude/CLAUDE.md — Global kurallar, bir kere kur 
# Global Claude Code Kuralları — Aykut
# Bu dosya tüm projelerde geçerli

## Dil ve İletişim
- Tüm yanıtları Türkçe ver
- Yorumları Türkçe yaz
- Hata mesajlarını Türkçe açıkla

## Kod Kalitesi — Asla Taviz Verme
- TypeScript: `any` tipi YASAK, her zaman doğru tip
- `console.log` bırakma, logger kullan
- Fonksiyonlar tek sorumluluk taşısın
- Magic number kullanma, sabit tanımla
- Yorumlar "ne yaptığını" değil "neden yaptığını" açıklasın

## Güvenlik
- .env dosyasına asla dokunma
- Şifreleri ve API anahtarlarını koda yazma
- Input validation her zaman yap

## Git Disiplini
- Test olmadan commit atma
- Commit mesajları Conventional Commits formatında
- Her commit tek bir iş yapmalı

## Çalışma Stili
- Bir şeyi değiştirmeden önce bana sor
- Emin olmadığında varsayım yapma, sor
- Büyük değişiklikleri küçük parçalara böl
- Her adımda ne yaptığını kısaca açıkla
// ADIM ADIM KURULUM

Nasıl kurulur? (Terminal bilmesen de olur)

Terminali aç (Mac'te: Cmd+Space → "Terminal" yaz → Enter). Aşağıdaki komutları sırayla yapıştır.

1

Klasörleri oluştur

Global Claude klasörü ve komut klasörü. Bir kere yap, sonsuza kadar kullan.

Terminal'e yapıştır 
mkdir -p ~/.claude/commands
2

Global CLAUDE.md dosyasını oluştur

Üstteki "Global Kurallar" kodunu kopyala, aşağıdaki komutu çalıştır, açılan editöre yapıştır, kaydet (Ctrl+X → Y → Enter).

Terminal'e yapıştır 
nano ~/.claude/CLAUDE.md
3

Global settings.json (Hooks) oluştur

Üstteki "Global Hooks" kodunu kopyala, aynı şekilde yapıştır ve kaydet.

Terminal'e yapıştır 
nano ~/.claude/settings.json
4

Komutları ekle

commit.md, review.md, deploy.md dosyalarını oluştur. Her birini ayrı ayrı oluşturup içine ilgili kodu yapıştır.

Terminal'e yapıştır 
nano ~/.claude/commands/commit.md
# Kodu yapıştır, Ctrl+X → Y → Enter
nano ~/.claude/commands/review.md
# Kodu yapıştır, Ctrl+X → Y → Enter
nano ~/.claude/commands/deploy.md
# Kodu yapıştır, Ctrl+X → Y → Enter
5

Her yeni projede: CLAUDE.md oluştur

Projenin ana klasörüne gir (index.js ya da package.json'ın olduğu yer), proje CLAUDE.md'ini oluştur ve proje bilgilerini doldur.

Her proje için — proje klasöründeyken yap 
# Proje klasörüne gir
cd ~/Projects/benim-projem

# CLAUDE.md oluştur
nano CLAUDE.md
# Üstteki proje CLAUDE.md şablonunu yapıştır, düzenle, kaydet
6

Claude Code'u yeniden başlat

Hook değişiklikleri için Claude Code'u kapatıp açman gerekiyor. CLAUDE.md değişiklikleri anında geçerli.
